# Spec 19: Verification & Telemetry API (Agent-Driven)

## Philosophy: Keystroke-Driven, Visual-Verified (KDVV)

The ExoTalk verification model prioritizes deterministic, programmatic interactions over manual GUI manipulation. By replacing mouse actions with global keyboard shortcuts (as established in [Walkthrough 24](../walkthroughs/24_keyboard_first_automation_and_test_stabilization.md)), we eliminate the fragility of coordinate-based automation while retaining full visual auditing capability.

## 1. Automated Testing Protocol

The [Agent Control Harness](../walkthroughs/41_connectivity_and_automated_testing.md) defines a three-tier verification loop for all UI-level scenarios.

### Tier 1: Precision Injection (Action)
- **Tool**: `xdotool`
- **Method**: Triggering global keyboard shortcuts (e.g., `Ctrl+M` for sidebar toggle, `Ctrl+1` for identity switch).
- **Rationale**: Short-circuits the need for mouse focus and pixel-perfect clicking, ensuring tests are robust against resolution scaling.

### Tier 2: State Inquiry (Milliseconds Verification)
- **ExoTalk (App)**: Inquire internal state via the **Telemetry API (Port 11434)**.
- **Conscia (Node)**: Inquire node status via the REST API (`/api/stats`, `/api/logs/stream`) over [SSH Tunnels](../walkthroughs/25_federated_access_and_security_fluidization.md).
- **Goal**: Sub-500ms confirmation of functional success before proceeding to the next step.

#### 1.1 Telemetry API (Local Sidecar)
The ExoTalk application runs a lightweight, read-only HTTP server on `127.0.0.1:11434` specifically for agent-driven verification and high-density UI updates.

| Endpoint | Data Returned | Purpose |
| :--- | :--- | :--- |
| `GET /api/identity` | Active DID, Name, Vault Status, Secret key hash | Verify identity synthesis and vault integrity. |
| `GET /api/network` | Node ID, Local IP:Port, Relay URL, Peer count | Verify mDNS/Relay connectivity and NAT state. |
| `GET /api/mesh` | Ingress/Egress throughput, Blob store size, Clock drift | Verify Willow/Iroh performance and sync state. |
| `GET /api/system` | OS/Arch, PID, Uptime, Memory usage | Verify deployment environment and process health. |

### Tier 3: Visual Confirmation (Audit)
- **Tool**: `scrot` / `browser_subagent`
- **Method**: Capture a high-resolution screenshot immediately following a successful Tier 2 inquiry.
- **Cross-Machine Capture (Exonomy)**: 
    - **Native UI**: Record the Exonomy desktop via the [RDP Bridge](../walkthroughs/25_federated_access_and_security_fluidization.md).
    - **Conscia Dashboard**: Record the tunneled remote dashboard in a local browser tab.
- **Rationale**: Provides the human-readable evidence required for documentation and verifies that the UI actually *rendered* the expected state change.

## 2. Systematic Scenario Progression

Recording and verification must follow a logical hierarchy of user complexity to ensure absolute baseline stability before moving to advanced mesh dynamics.

### Phase 1: Fundamental UI & Identity
- **Account Management**: Launching with no identity → Onboarding Wizard → Identity Synthesis.
- **Vault Governance**: Switching identities, "Danger Zone" wipes, and recovery verification.
- **Status Awareness**: Theme toggling, UI scaling, and sidebar responsiveness.

### Phase 2: Peer-to-Peer Dynamics
- **Identity Pairing**: QR-code based pairing and manual token transfer.
- **Gossip Chat**: Sub-second message delivery between Exocracy and Exonomy in a pure P2P (tunnel) environment.

### Phase 3: Node Management & Distribution
- **Node Association**: Establishing a Lifeline connection to a local/remote Conscia node.
- **Cross-Machine Deployment**: Using ExoTalk on Exocracy to deploy a fresh Conscia node to Exonomy.
- **Distribution Parity**: Verify and record each [Conscia Distribution Method](conscia_distribution.md) (Apt, NPM, Cargo, Docker) to ensure the daemon initializes correctly across all paths.

## 3. Documentation & Tutorial Pipeline

This verification protocol serves a dual purpose: it validates the codebase and generates the assets for the [Screenplays](../scenarios/screenplays/README.md).

### 3.1 Asset Collection
- **Split-Screen Recordings**: Synchronized capture of Exocracy and Exonomy UI states to demonstrate real-time P2P coordination.
- **Static Screenshots**: Tier 3 screenshots are preserved and automatically linked in the [Walkthroughs](../walkthroughs/README.md) to provide a non-interactive, step-by-step tutorial experience.

### 3.2 AI-Generated Video Production
- **Visuals**: Preserved video segments from the automated test runs.
- **Audio**: Narrative voice-over and musical soundtrack generated by AI based on the screenplay scripts.
- **Consistency**: Ensures the tutorial exactly matches the behavior of the current version of the software.
