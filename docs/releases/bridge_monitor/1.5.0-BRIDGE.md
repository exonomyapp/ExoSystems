# Exotech Bridge Monitor - Release `v1.5.0-BRIDGE`

**Date**: May 7, 2026
**Target**: Exonomy Node Deployment
**Binary Size**: 24,304 bytes

## Release Notes

This release supersedes the `v1.4.0-STATIC` and `v1.4.0-MOSES` internal builds. It formalizes the static, zero-overhead paradigm established during the efficiency profiling phase.

### Core Architecture
- **Event-Driven Static Indicators**: All background Rust polling and FragmentShader animations have been stripped out. The `_StatusIndicator` widget now returns simple, static Flutter primitives (`BoxDecoration` with solid colors).
- **Zero-Overhead Idle**: Eliminates the persistent CPU baseline floor by relying purely on `systemctl` telemetry sweeps driven by user interaction or interval polling, completely discarding continuous main-thread GIF/animation decoding.

### Diagnostic Visuals
The node cards provide static color indications:
- **Intense Neon Green**: Node is UP with active traffic.
- **Dark Unlit Green**: Node is UP but idle.
- **Static Yellow**: Node is SLEEPING.
- **Static Red**: Node is OFF.

### Deployment Status
- Shipped directly to Exonomy node (`~/deployments/bridge_monitor/bundle/exotech_bridge`).
- The corresponding `~/Desktop/exotech_bridge.desktop` shortcut points directly to this release bundle.
